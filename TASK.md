# Тестовое задание: быстрая смена ключей шифрования

## Цель задания

- Разработать сервис, обеспечивающий быструю смену ключей шифрования для набора бинарных данных, разбитых на чанки.

## Исходные условия

- Есть файл размером 80 ГБ, например фильм.
- Файл необходимо представить в виде массива байт.
- Полученные данные нужно разделить на чанки по 50 КБ.
- Каждому чанку должен быть присвоен уникальный UUID v7.
- Должны использоваться две независимые базы данных PostgreSQL.

## Хранение данных

- База данных 1 хранит UUID чанка и сгенерированный случайным образом AES-256 ключ для каждого чанка.
- База данных 2 хранит UUID чанка и чанк, зашифрованный соответствующим AES-256 ключом.

## Основная задача

- Необходимо реализовать API для смены ключей шифрования.
- Для каждого UUID чанка должен быть сгенерирован новый AES-256 ключ.
- Новый ключ должен быть записан в первую базу данных.
- Соответствующий чанк во второй базе данных должен быть перешифрован новым ключом.

## Дополнительная задача

- Необходимо реализовать второе API, которое расшифровывает чанки, собирает их в исходный файл и возвращает собранный файл без повреждения данных.

## Ожидаемый результат

- Максимальное быстродействие смены ключей.
- Максимальное быстродействие перешифрования чанков.
- Корректная сборка исходного файла после смены ключей.

## Критерии оценки

- Скорость смены ключей шифрования.
- Скорость перешифрования чанков.
- Корректность собранного после смены ключей файла.
