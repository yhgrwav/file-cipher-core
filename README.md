# file-cipher-core

Сервис быстрой смены ключей шифрования для бинарных данных, разбитых на чанки.

Файл режется на чанки по 50 КБ, каждый чанк получает UUID v7 и собственный AES-256-GCM
ключ. Данные хранятся в двух независимых PostgreSQL: **БД ключей** (`uuid → ключ`) и
**БД данных** (`uuid → зашифрованный чанк`). Курсоры ротации хранятся в Redis.

## Запуск

```
git clone https://github.com/yhgrwav/file-cipher-core
cd file-cipher-core
cp .env.example .env
docker compose up --build
```

Поднимутся обе БД, Redis, применятся миграции и запустится приложение на `:8080`.
Настройки — в `.env`.

## API

| Метод | Путь | Что делает |
|-------|------|------------|
| `POST` | `/files` | Принимает файл телом запроса, шифрует, возвращает `file_id` |
| `POST` | `/files/{fileID}/rotate` | Меняет ключи всех чанков файла |
| `GET`  | `/files/{fileID}` | Расшифровывает и отдаёт исходный файл |

## Проверка в Postman

### 1. Зашифровать

- Метод **POST**, URL `http://localhost:8080/files`
- Вкладка **Body** → **binary** → **Select File** → выбрать любой файл
- **Send**

В ответе — `file_id`, скопировать его:
```json
{ "file_id": "0191b2c3-...." }
```

### 2. Сменить ключи

- Метод **POST**, URL `http://localhost:8080/files/{file_id}/rotate`
- Body: **none**
- **Send** → ожидается `200 OK`

### 3. Скачать и проверить

- Метод **GET**, URL `http://localhost:8080/files/{file_id}`
- Нажать стрелку рядом с **Send** → **Send and Download**, сохранить файл

Скачанный файл должен побайтово совпадать с исходным. Проверка в PowerShell:
```powershell
Get-FileHash исходный, скачанный | Format-Table Hash
```

Порядок проверки: **зашифровать → скачать (сверить) → сменить ключи → скачать снова
(сверить)**. Совпадение хешей после ротации — файл цел, ключи сменены.
